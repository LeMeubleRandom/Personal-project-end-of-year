Résumé de l'architecture du projet Twisted Realms

1. Vue d'ensemble
-----------------
Twisted Realms est une application web de type jeu de cartes à collectionner (TCG) avec :
- un frontend React/Vite dans `client/`
- un backend Node.js/Express dans `server/`
- une base de données MySQL
- un service Redis pour cache/session ou autres usages
- une communication temps réel via Socket.IO

2. Frontend
-----------
Emplacement : `twisted-realms/client/`
Technologies principales : React 19, Vite, React Router, Socket.IO client

Structure principale :
- `index.html` : page d'entrée HTML
- `src/main.jsx` : point d'entrée JS qui monte l'application React
- `src/App.jsx` : composant principal de l'application
- `src/index.css` : styles globaux
- `src/components/` : composants partagés
  - `AccountForm.jsx`, `Footer.jsx`, `GlobalChat.jsx`, `Header.jsx`, `NotificationForm.jsx`, `SecurityForm.jsx`
- `src/views/` : pages et vues principales
  - `Collection.jsx`, `Home.jsx`, `Lobby.jsx`, `Login.jsx`, `NotFound.jsx`, `Profile.jsx`, `Register.jsx`, `Shop.jsx`
- `public/default-images/` et `public/user-images/` : ressources et images statiques

Dépendances importantes :
- `react`, `react-dom`
- `react-router-dom`
- `socket.io-client`

3. Backend
-----------
Emplacement : `twisted-realms/server/`
Technologies principales : Node.js, Express, MySQL, Redis, Socket.IO, JWT, bcrypt, multer

Entrée principale :
- `app.js` : configuration du serveur Express, CORS, middleware, routes et Socket.IO

Organisation du backend :
- `controller/` : logique de contrôle des requêtes
  - `AdminController.js`, `CardController.js`, `CollectionController.js`, `DeckController.js`, `MessageController.js`, `UserController.js`
- `router/` : routes Express par domaine
  - `MessageRouter.js`, `UserRouter.js`
- `service/` : services métiers
  - `UserService.js`
- `model/` : modèles de données
  - `Message.js`, `User.js`
- `db/` : configuration des bases de données
  - `mysql.js` : connexion MySQL
  - `redis.js` : connexion Redis
- `middleware/` : middlewares spécifiques
  - `auth.js` : gestion de l'authentification
  - `multer.js` : configuration des uploads de fichiers
- `script/` : logique de jeu
  - `game.js`

Dépendances importantes :
- `express`, `cors`, `cookie-parser`, `dotenv`
- `mysql`, `mysql2`
- `redis`
- `jsonwebtoken`, `bcrypt`
- `multer`
- `socket.io`

4. Architecture logicielle
--------------------------
- Frontend React consomme le backend via API REST (`/user`, `/message`, etc.).
- Backend expose des routes API gérées par des routers et controllers.
- L’authentification est gérée par JWT et cookies.
- Redis est présent comme stockage secondaire ou cache.
- Socket.IO gère le chat en temps réel et la communication instantanée.
- Les images uploadées sont servies en statique depuis `server/public/user-images/`.

5. Déploiement / exécution
--------------------------
- Frontend : `npm run dev` dans `twisted-realms/client/`
- Backend : `npm start` dans `twisted-realms/server/` (avec `nodemon`)
- Le backend écoute sur le port `5000` par défaut.

6. Points clés
--------------
- Architecture séparée frontend/backend
- React + Vite côté client
- Node.js + Express côté serveur
- Base de données MySQL + Redis
- Temps réel avec Socket.IO
- Authentification et gestion de fichiers via middleware
